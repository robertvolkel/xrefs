Key insight: Claude decides which tools to call and in what order. It may search, inspect attributes, ask the user clarifying questions, then find replacements — all within a single conversation turn. The tool loop continues until Claude has enough information to provide an assessment.

Matching Engine Pipeline

Inside getRecommendations() — from source part to ranked results

Evaluate every rule
calculate matchPercentage
↑ XrefRecommendation[]
Scoring formula: matchPercentage = (earnedWeight / totalWeight) × 100. A candidate fails if any rule returns fail. Application review rules earn 50% credit; operational mismatches earn 80% credit. Post-scoring filters remove hard-gate mismatches (e.g., wrong architecture, wrong protocol) that can't be caught by weight-based scoring alone.

A typical flow: search_parts → find the right MPN → get_part_attributes → review specs → find_replacements → analyze scored results → respond with assessment. The loop may iterate 2-4 times per query. If the user asks a follow-up, filter_recommendations narrows the existing results without re-running the full pipeline.
